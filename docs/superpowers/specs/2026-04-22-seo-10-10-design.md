# SEO 10/10 Design — OddsDetector

**Date:** 2026-04-22  
**Goal:** Bring OddsDetector from ~6.5/10 to 10/10 SEO maturity.  
**Brand:** OddsDetector (default site URL: `https://www.oddsdetector.com`)

---

## Audit Summary

The codebase already has solid Next.js 16 metadata API usage, structured data on fixture/team/standings pages, robots.txt, and sitemap. The gaps are specific:

| Priority | Issue |
|----------|-------|
| P0 | Root description says "admin-controlled betting content" — shows in SERP for every page without its own description |
| P0 | `app/bonus-codes/page.tsx` is `'use client'` with zero metadata, zero schema — page is in sitemap and publicly indexed |
| P0 | OG `title` fields manually append `" \| OddsDetector"` on pages where Next.js title template already adds it — causes double-brand in social previews |
| P1 | `tennis/` and `cs2/` are thin "coming soon" pages with no `noindex` — crawl waste |
| P1 | No root `WebSite` or `Organization` JSON-LD — Google can't build brand knowledge panel |
| P1 | No OG image anywhere — social shares render blank preview card |
| P1 | `SportsEvent.eventStatus` uses internal `stateBucket` string, not schema.org `EventStatusType` URI |
| P1 | `SportsEvent.location.address` is a plain city string — schema.org requires a `PostalAddress` object |
| P2 | Sitemap `lastModified` is always `new Date()` (build time) — not semantically meaningful |
| P2 | `app/football/leagues/page.tsx` just redirects — already excluded from sitemap, fine |
| P2 | No `docs/seo-plan.md` for ongoing execution |

## Detailed Specs

### 1. Root layout — description + global JSON-LD

**`app/layout.tsx`**

Replace description:
```
'Football odds comparison, live market tracking, and admin-controlled betting content.'
```
With:
```
'Compare football odds across bookmakers, track live markets, and find the best prices on OddsDetector.'
```

Add two JSON-LD blocks via `<script type="application/ld+json">` inside `<head>`, rendered server-side (not via next/script — keep it synchronous for crawlers):

**WebSite** (enables sitelinks searchbox potential):
```json
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "@id": "https://www.oddsdetector.com/#website",
  "name": "OddsDetector",
  "url": "https://www.oddsdetector.com",
  "description": "Football odds comparison and live market tracking",
  "potentialAction": {
    "@type": "SearchAction",
    "target": {
      "@type": "EntryPoint",
      "urlTemplate": "https://www.oddsdetector.com/?q={search_term_string}"
    },
    "query-input": "required name=search_term_string"
  }
}
```

**Organization**:
```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "@id": "https://www.oddsdetector.com/#organization",
  "name": "OddsDetector",
  "url": "https://www.oddsdetector.com",
  "logo": {
    "@type": "ImageObject",
    "url": "https://www.oddsdetector.com/icon"
  },
  "sameAs": []
}
```

Both use `getSiteUrl()` dynamically, not hardcoded strings. Build the JSON in the server component function and render via `dangerouslySetInnerHTML`.

Since `RootLayout` is currently a server component (no `'use client'`), add the scripts inside `<head>` before the existing `<Script>` theme-init tag.

### 2. OG image default

**`app/opengraph-image.tsx`** — Next.js App Router auto-discovery. When present at `app/opengraph-image.tsx`, it serves as the default `og:image` for every page that does not define its own.

Use `ImageResponse` from `next/og`. Size: 1200×630 (standard OG).

Design: Dark background (`#07101a`), OddsDetector logo/wordmark centred, green accent (`#00e676`), tagline "Football Odds Comparison". Keep it simple — it's a fallback card, not a per-page image.

Export `alt`, `size`, and `contentType` as required by Next.js conventions.

### 4. Fix OG title double-branding

On all pages that manually do `title: \`${title} | OddsDetector\`` in their OG/twitter fields, remove the ` | OddsDetector` suffix. Next.js applies the root `template: '%s | OddsDetector'` to the `<title>` tag only — OG title is set explicitly and does NOT go through the template, so manually appending it is correct behaviour for `og:title`. **However**, several pages set it inconsistently. The rule:

- `og:title` should be the full readable title that will appear in social cards. Include brand suffix once: `"Premier League Live Odds & Fixtures | OddsDetector"`.
- Pages that already set it correctly (with one suffix): leave as-is.
- Pages that produce a double suffix (e.g. fixture page where `buildFixtureMetadataTitle` already returns a clean title and then OG field appends ` | OddsDetector` redundantly when the template would also add it to `<title>`): verify carefully. The template only affects `<title>`, not `og:title`, so the OG title suffix is intentional and correct. **Do not remove it.** What needs fixing is any page where the string being set already contains ` | OddsDetector` before the OG field appends another one.

On review: fixture page title is `"Foo vs Bar Odds & Match Details"` → OG title becomes `"Foo vs Bar Odds & Match Details | OddsDetector"` → correct. No change needed there.

**Real issue**: the root layout description. That's the only double-impact fix here. The OG title pattern is actually fine as-is on existing pages.

### 5. SportsEvent schema corrections

**`app/football/fixtures/[fixtureId]/page.tsx`**

Fix `eventStatus`: Map `stateBucket` to schema.org EventStatusType URI:
- `'Live'` → `'https://schema.org/EventScheduled'` (in-progress; no "EventInProgress" in schema.org — use EventScheduled for live, or omit if uncertain)
- `'Upcoming'` → `'https://schema.org/EventScheduled'`
- `'Finished'` → `'https://schema.org/EventCompleted'`
- `'Postponed'` → `'https://schema.org/EventPostponed'`
- `'Cancelled'` → `'https://schema.org/EventCancelled'`
- anything else → omit the field

Fix `location.address`: Change from plain string to PostalAddress:
```json
{
  "@type": "PostalAddress",
  "addressLocality": "CityName"
}
```

Add `organizer` field pointing to the league as SportsOrganization.

Add `description` field using the same text as the page meta description.

### 6. Tennis + CS2 noindex

**`app/tennis/page.tsx`** and **`app/cs2/page.tsx`**:

```ts
import type { Metadata } from 'next';

export const metadata: Metadata = {
  title: 'Tennis Odds — Coming Soon',
  robots: { index: false, follow: false },
};
```

This prevents crawl waste and signals to Googlebot these are placeholder pages.
