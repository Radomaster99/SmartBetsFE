Quick wins / polish
🔝 Topbar
🎁
Bonus Codes button is visually dominant
Topbar.tsx — both mobile & desktop
The gold pulsing "🎁 Bonus Codes" button competes with the primary navigation — it's styled more prominently than search and the live counter, and it pulses constantly. It feels like a banner ad placed in the nav, not a navigation item. Bettors find this irritating.

Fix: Move to a subtle text link or a secondary chip. Let search and the live counter be the visual anchors.
● High
🔍
Search has no placeholder context
GlobalSearch — desktop only
The search bar is visible on desktop but the placeholder doesn't tell users what they can search. Heavy users won't explore — they need confirmation the feature does what they expect.

Fix: Placeholder text: "Search teams, leagues, fixtures…". Also consider keyboard shortcut hint like ⌘K on the right edge.
● Medium
🌗
No theme toggle in the topbar
Topbar.tsx — desktop
Light/dark toggle exists via ThemeContext and is stored in localStorage, but the control is nowhere in the main nav on desktop. Users can't find it without hunting.

Fix: Add a small sun/moon icon button in the topbar right side. 1 minute of code, high discoverability impact.
● Medium
🔴
Live counter disappears when count is zero
Topbar.tsx — desktop
The live badge only renders when liveCount > 0. This causes layout shift when fixtures go live during a session, and the topbar feels inconsistent.

Fix: Show a dimmed "0 Live" badge permanently. When count increases, animate the number in.
● Low
🗂 Sidebar — League Navigation
📌
No visual hierarchy — pinned, popular, and all-leagues look the same
FootballSidebarContent.tsx
All league navigation items are rendered at the same visual weight. Pinned leagues are labeled with a tiny "PINNED" header but items themselves are indistinguishable from regular leagues. Country group sections blend with everything else.
Current
All items same size, weight, and color. "PINNED" label in small uppercase but no visual distinction per item. Country groups presented identically to section headers. No live fixture count per league.
Improved
Pinned items show a ★ pin icon. Popular leagues have subtle green-tinted background. Country groups collapse by default with an expand arrow. League rows show a live dot + count when that league has live fixtures. Active league gets accent border-left.
● High
🔴
No live indicator per league in the sidebar
FootballSidebarContent.tsx
When a league has live matches right now, there's zero indication in the sidebar. A bettor following La Liga has to navigate there to discover there's a live game happening — they won't know unless they're already on that view.

Fix: Attach a red dot + count badge to league items when live fixtures exist for that league. This data is available client-side.
● High
📐
Sidebar fixed at 224px — tight for long league names
Sidebar.tsx — md:w-56
224px is quite narrow for leagues with long names (e.g. "Primera Division RFEF Group 2"). Names get truncated with ellipsis. On xl+ screens this leaves unused space in the main content area while the sidebar feels cramped.

Fix: Bump to 240–256px on xl screens with xl:w-64. Or make it resizable/collapsible.
● Low

● High
👆
Odds buttons are 32px tall — too small on touch
OddsButton in FixtureRow.tsx
The three odds buttons (HOME/DRAW/AWAY) are height: 32px. Apple HIG recommends 44px minimum for touch targets. On a 390px-wide phone, tapping the correct bookmaker odds is error-prone. The bookmaker label at 7px font is also unreadably small — should be 9–10px minimum.

Fix: Increase button height to 40–44px. Bookmaker name to 9px.
● High
‹›
Date nav arrows are plain ASCII < > text
FixtureFilters.tsx — all breakpoints
The previous/next day buttons render as literal < and > characters. They look unfinished and are very small click/tap targets with no visual affordance.

Fix: Replace with SVG chevron icons (16×16px) with adequate padding to hit 36×36px minimum touch area.
● Medium
⭐
Watchlist button uses raw Unicode ☆/★ glyphs
FixtureRow.tsx — save button
The watchlist toggle uses '★' and '☆' Unicode characters. These render inconsistently across browsers and OS (different sizes, different weights, sometimes emoji vs text style).

Fix: Replace with a proper SVG bookmark or heart icon for consistent cross-platform rendering.
● Low
📊
Odds movement arrows are 9px and invisible at a glance
OddsButton — movement indicator
Odds movement direction uses ↑/↓ at 9px font, positioned in the top-right corner of each odds button. These are extremely hard to see during fast scanning, especially while the flash animation is also running.

Fix: Use 12px colored triangle/chevron SVG. Add a brief colored background pulse on the button itself (not just a corner indicator).
● Medium
💰 Odds Panel & Fixture Detail
🏢
Bookmaker rows have no logos — only 2-letter initials
OddsComparison.tsx / BestOddsBar.tsx
Bookmaker identity relies on a colored circle with a 2-letter abbreviation (e.g. "B3" for Bet365). Heavy bettors recognize bookmakers by logo instantly — the current system requires reading the name text on every scan.

Fix: Add 20×20px brand favicon or SVG logo for top bookmakers. Even just better icon design (distinct shapes, not just colors) would help.
● Medium
📈
No odds history sparkline
Fixture detail — Odds tab
The odds table shows current prices but no historical movement. Line movement is critical signal for sharp bettors — is this price opening or closing? A tiny 40px sparkline next to each row showing the last 6–8 data points would be a major differentiator vs competitors.
● Medium
‸
Best-odds cards use ^ and v as movement arrows
OddsComparison.tsx — bestOutcomes grid
The best-odds header cards show ^ and v (keyboard caret/letter) as up/down movement indicators. These are especially confusing — v looks like a letter, not a down arrow.

Fix: Use proper ↑/↓ Unicode arrows or SVG chevrons, with color: green for up, red for down.
● Medium
🎯
BET button needs stronger visual prominence
BestOddsBar.tsx — OddsRow
The "BET" CTA is a primary conversion action but uses a generic cta-btn class and doesn't visually stand out from the row. It should be the most visually dominant element per row: bold green fill, high contrast white text, consistent 36–40px height, slight hover lift effect.

The "Open" button in compact variant is even weaker — undersized and misnamed (should say "BET" consistently).
● High
📱 Mobile Experience
📱
Fixture row layout is too cramped at 375–390px
FixtureRow.tsx — mobile widths
The current single-row layout packs status + home team + score + away team + 3 odds buttons into one horizontal band. On a 390px iPhone, team names truncate to 4–5 characters and odds buttons are 32px × ~60px each — barely tappable.
Current (single row)
[20:00] [Manch...] vs [Liverp...] [2.15↑ B365][3.40 BB][3.10 B365][☆]

Everything in one cramped horizontal band. Team names get aggressive truncation.
Improved (two rows)
Row 1: [20:00] Manchester City vs Liverpool (full names)
Row 2: [HOME 2.15 B365↑] [DRAW 3.40 BB] [AWAY 3.10 B365] full width, taller buttons

Cleaner, more tappable, teams fully visible.
● High
⬇️
Bottom nav inactive tabs at 28% opacity look broken
MobileBottomNav.tsx
Inactive tabs use opacity: 0.28 with a grayscale(1) filter. This makes the nav look visually impaired or disabled — like a design placeholder. Standard mobile nav uses 55–65% opacity for inactive items.

Fix: Change to opacity: 0.5, remove grayscale filter. Labels at 10px are borderline — try 11px.
● Medium
👆
Date strip has no swipe gesture support
FixtureFilters.tsx — mobile
The date strip on mobile navigates via prev/next buttons. Users expect to swipe left/right on the date row to change days — this is a standard gesture on every sports app (Flashscore, OneFootball, Google Calendar).

Fix: Add a touch event handler on the strip container that detects horizontal swipe and calls onDateChange.
● Low
🎯 Conversion & Referral
💵
No hover feedback or CTA tooltip on bookmaker odds buttons
OddsButton in FixtureRow.tsx
Clicking a bookmaker odds button opens the bookmaker in a new tab — that's good. But there's no hover state, no tooltip, and no click confirmation. A bettor doesn't know where they're going until they land there.
Current
Odds button shows number + 7px bookmaker name. No hover state. No tooltip. Click opens new tab silently. No visual confirmation that something happened.
Improved
On hover: brighter border + tooltip "Bet 2.15 at Bet365 ↗". On click: brief scale-down + checkmark flash. Bookmaker name 9–10px. External link icon (↗) on the button signals it opens a new tab. Builds trust before the click.
● High
🏷️
Bonus codes page is not surfaced on mobile bottom nav
MobileBottomNav.tsx
The bonus codes page (a conversion-critical page) is only reachable from the topbar on mobile — a gold pulsing button that many users may dismiss as an ad. The mobile bottom nav has 4 tabs (Matches, Search, Leagues, Saved) with no path to bonus codes.

Fix: Consider adding a "Offers" or "Bonuses" 5th tab on mobile, or surfacing it more prominently in the Leagues sheet.
● Medium
🎨 Visual Polish
🖼️
Empty states are plain text — no illustration
EmptyState component
When no fixtures match the active filter (e.g. no live games, no games for selected date), the empty state shows only text. This feels like an error rather than a designed state.

Fix: Add a small inline SVG illustration (football, binoculars, calendar) and a contextual sub-message like "No live fixtures right now — check upcoming games."
